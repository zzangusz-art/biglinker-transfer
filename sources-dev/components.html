FAQ Item

details/summary 기반 아코디언. 화살표 rotate 애니메이션 포함. white 배경 전용.

코칭은 몇 회로 구성되나요? ▾

기본 3회 코칭(구조 설계 → 내용 심화 → 최종 완성)으로 구성됩니다. 진도와 완성도에 따라 1~2회 추가가 가능합니다.

코칭은 어떤 방식으로 진행되나요? ▾

화상 미팅 또는 문서 피드백 방식 중 선택 가능합니다. 각 세션은 60~90분 내외로 진행됩니다.

어느 시점에 시작하는 것이 좋나요? ▾

마감 최소 4주 전을 권장합니다. 급한 경우 마감 2주 전도 가능하나 집중 세션으로 진행됩니다.

COACHING PROCESS

기본 3회 코칭 으로
문항 구조부터 최종 제출본까지 정리합니다

초안이 없어도 방향 설정부터 시작할 수 있습니다.
이미 초안이 있다면 구조 보완과 내용 밀도 강화에 집중합니다.

01
1회차 · 방향 설계

초안 작성 및 전략 방향 설정

지원자의 경험, 연구 관심사, 목표를 심층 분석해 핵심 키워드를 도출하고 문서 전체의 서사 구조를 먼저 설계합니다. 지원 학교·교수 연구 분야 접점도 이 단계에서 파악합니다.

02
2회차 · 내용 심화

피드백 및 내용 수정

논리적 흐름, 차별화 포인트, 학교·학과 적합성을 점검하며 구체적인 사례와 표현을 보완합니다. 구체성이 없는 서술은 실제 경험 기반으로 교체하고, 연구 필요성과 진로 비전을 강화합니다.

03
3회차 · 최종 완성

최종 첨삭 및 제출본 마무리

문장 완성도, 오탈자, 분량, 제출 형식을 최종 점검하고 실제 제출 가능한 수준까지 다듬습니다. 진도에 따라 1~2회 추가 코칭이 이어질 수 있습니다.

구성 요소
  • .process-head — 섹션 제목 + 설명 (max-width 720, center)
  • .timeline — gap 18px grid 컨테이너
  • .timeline-item — 72px step icon + body 2열 grid. ::before conic 테두리 애니메이션
  • .timeline-step — 72×72 아이콘 박스
  • .step-star — 우상단 별 장식 SVG (absolute)
  • .timeline-meta — "N회차 · 단계명" 상단 레이블

Text Emphasis (.em)

헤딩 텍스트 내 핵심 단어에 SVG 손그림 밑줄을 얹는 강조 패턴. 선 굵기·색상·경로로 느낌이 달라진다.

단일선 — dark bg

자기소개서와 학업계획서를 합격 기준으로 다시 설계합니다

단일 얇은선 — brand-2

서류는 단순한 글이 아니라 지원자의 미래를 설계하는 문서입니다

이중선 굵게 — light bg

합격 서류와 탈락 서류의 차이는 표현이 아니라 구조입니다

짧은 단어 강조 (process heading)

기본 3회 코칭 으로 완성합니다

사용 패턴

구조
<span class="em"> 강조 텍스트 <svg>…</svg> </span>

SVG viewBox
텍스트 길이에 맞춰 width를 조정. 짧은 단어 → 100~130, 긴 문장 → 200~300. height는 항상 14 고정.

선 종류
단일선: path 1개, stroke-width 2.4~2.8
이중선: path 2개 (굵은선 opacity 0.85 + 얇은선 opacity 0.4~0.45)
색상: dark bg → #4a68ff 또는 #7f92ff / light bg → #4a68ff

적용 규칙
h1, h2 헤딩에만 사용. 한 헤딩에 한 군데만 강조. 본문(p)에는 사용하지 않는다.
