BigLinker 사이트 제작 조건
1. 디자인 시스템
기준 파일: components.html + design.md
색상, 레이아웃, 컴포넌트 패턴은 이 두 파일에서만 가져옴
이미지를 첨부해서 "이렇게 만들어달라"고 해도 내용(텍스트/구조)만 참고, 색상과 레이아웃은 무시
2. 배경색 표준
다크 우선: body는 항상 다크 그라데이션 (#090b0f → #0d1015)
섹션 구성: 대부분 section-dark → 2개 내외만 section-light / section-panel (시각적 리듬용)
Hero도 다크 (배경 없이 body 상속)
3. 명도 대비 규칙
배경 결정 → 내부 텍스트 색상 반드시 대비 확인
section-dark 안에 흰색 카드(background:#fff)가 있다면 반드시 카운터 리맵 클래스 부여
다크 섹션: --header-text → var(--text), --header-muted → var(--muted) 자동 리맵
흰 카드 안: --header-text: #17191f, --header-muted: #6e7687 복구
4. 헤더
self-introduction.html의 헤더(top-shell + sticky-category-shell) CSS/HTML을 그대로 복사
반응형 760px 브레이크포인트 포함, JS 스크롤 콜랩스 포함
어떤 새 페이지든 예외 없이 동일 적용
5. 컴포넌트 패턴
리뷰 섹션 → review-band 마퀴 (3행 무한 스크롤)
FAQ → faq-band (다크 bg, details/summary)
CTA → cta-band (다크 + radial gradient)
카드 → .eval-card, .phase-step, .session-card, .principle-card 등 기존 클래스 재사용
